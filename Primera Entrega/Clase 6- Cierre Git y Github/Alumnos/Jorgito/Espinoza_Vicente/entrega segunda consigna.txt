git clone <https://link-con-nombre-del-repositorio>
realiza una copia idéntica de la última versión de un proyecto en un repositorio y la guarda en tu ordenador.

git branch <nombre-de-la-rama>
creará una rama en local.

git push <nombre-remoto> <nombre-rama>
Para enviar (push) la nueva rama al repositorio remoto.

git checkout <name-of-your-branch>
Usamos git checkout principalmente para cambiar de una rama a otra. 

git status
nos brinda toda la información necesaria sobre la rama actual.

git add <file>
para incluir los cambios de un archivo(s) en nuestro próximo commit.

git commit -m "commit message"
Git commit es como establecer un punto de control en el proceso de desarrollo al que puede volver más tarde si es necesario. También necesitamos escribir un mensaje corto para explicar lo que hemos desarrollado o cambiado en el código fuente.

Git push
Después de confirmar los cambios (con git commit), lo siguiente que hay que hacer es enviar estos cambios al servidor remoto. Git push sube tus confirmaciones al repositorio remoto.

git pull <remote>
El comando git pull se usa para obtener actualizaciones del repositorio remoto. En simples palabras, sirve para traer el repositorio remoto a tu repositorio local.
